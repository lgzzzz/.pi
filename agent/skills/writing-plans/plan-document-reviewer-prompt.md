# 计划文档评审员提示词模板

使用`junie_ai`工具评审规范计划并且使用此模板。如果`junie_ai`工具报错,使用subagent `worker`.

**目的：** 验证计划是否完整、是否符合规范，以及任务分解是否恰当。

**派遣时间：** 完整计划编写完成后。

```
  提示词: |
    验证此计划是否完整并已准备好进行实施。

    **待评审计划:** [PLAN_FILE_PATH]
    **参考规范:** [SPEC_FILE_PATH]

    ## 检查内容

    | 类别 | 检查重点 |
    |----------|------------------|
    | 完整性 | TODO、占位符、不完整的任务、缺失的步骤 |
    | 规范一致性 | 计划涵盖了规范要求，无明显的范围蔓延 |
    | 任务分解 | 任务边界清晰，步骤具有可操作性 |
    | 可构建性 | 工程师能否在不卡壳的情况下遵循此计划？ |
    | 规范覆盖率 | 快速浏览规范中的每个部分/要求。你能指出实现它的任务吗？列出任何遗漏？ |
    | 类型一致性 | 你后续任务中使用的类型、方法签名和属性名称是否与在之前任务中定义的相匹配？例如，在任务 3 中称为 `clearLayers()` 的函数在任务 7 中变成了 `clearFullLayers()`，这就是一个 bug。 |

    ## 校准标准

    **仅标记会在实施过程中导致实际问题的缺陷。**
    实施者构建了错误的内容或卡壳是问题。细微的用词改进、风格偏好和“锦上添花”的建议则不是。

    除非存在严重缺陷 —— 如遗漏规范要求、相互矛盾的步骤、占位符内容或过于模糊以至于无法执行的任务，否则请予以通过。

    ## 输出格式

    ## 计划评审报告

    **状态:** 已批准 | 发现问题

    **问题 (如有):**
    - [任务 X, 步骤 Y]: [具体问题] - [对实施的影响]

    **建议 (参考性，不阻碍批准):**
    - [改进建议]
```

**评审员返回：** 状态、问题 (如有)、建议
